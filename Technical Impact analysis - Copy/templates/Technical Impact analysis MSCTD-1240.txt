What's the Analysis?
---------------------

1. Quotation No.,Customer Order No.,Salesinvoice No. Format to be corrected

   Analysis:
   --------
   
   -> Parts Sales, the Quotation, Customer Order, and Proforma Invoice documents follow the Financial Year format and will display the fiscal year as 26–27. However, the Sales Invoice numbering is configured differently and uses only the ending year of the financial period, meaning invoices generated during FY 2026–27 will carry 27 in the numbering sequence.
   
   -> For this requirement, the sequence table also needs to be reviewed and updated. Currently, the sequence values are maintained based on the existing functionality, where the Sales Invoice numbering follows the financial year-end format (26–27) and the sequence is updated accordingly for the years 26 and 27.
   
   -> If we change the logic now to use the current calendar year format, the system will create a new reference record with the year value 26 for the current year. As a result, invoices generated before March (within the current financial year) will continue to pick the 26 sequence value based on the updated sequence configuration. Therefore, the impact on the existing sequence records and primary key generation needs to be analyzed before implementing this change to avoid duplicate references or incorrect invoice numbering.

   Existing Solution:
   -----------------
   
   -> In the existing numbering format for Parts Sales, the Quotation, Customer Order, and Proforma Invoice documents follow the Financial Year format and will display the fiscal year as 26–27. However, the Sales Invoice numbering is configured differently and uses only the ending year of the financial period, meaning invoices generated during FY 2026–27 will carry 27 in the numbering sequence.

   Proposed Solution:
   ------------------

   -> For Parts Sales, the Quotation, Customer Order, and Proforma Invoice numbers will continue to follow the Financial Year format and display the fiscal year as 26–27. However, the Sales Invoice number will be generated based on the current calendar year. For example, if the invoice date is 25/06/2026, the Sales Invoice number will carry 26 as the year identifier instead of the fiscal year-end value (27).
   
   
2. Item Rate Change Screen Enhancements
	
	Analysis:
    --------
   
    -> Existing screen was working as expected. However, after the recent upgrade for the new LiuGong implementation, there appears to be a change in the behavior. Previously, the system accepted only the Effective From Date, and users could add records only for the next day. Now, users are able to add an item with the current date itself, but the effective time is automatically set to 3 hours later instead of the current time. This change in functionality needs to be reviewed and confirmed.	

    Existing Solution:
    -----------------
   
    -> Existing screen was working as expected. The system allowed users to enter the Effective From Date only for the next day. The date restriction was applied as per the existing functionality, and the process was working without any issues.

    Proposed Solution:
    ------------------

    -> As part of the recent upgrade for the new LiuGong implementation, the functionality has been enhanced to allow users to add items from the current date itself. However, the system is now setting the effective time automatically to 3 hours later than the current time.
